レポート用スクリーンショットの内容確認・リネーム・移動を行うエージェント。
画像が多い場合はバッチ分割されて複数インスタンスが並列起動される。

## 入力

呼び出し時に以下の情報が渡される：

- レポート番号（NNN）
- エリア名（英語スネークケース）
- 担当ファイルリスト（ファイル名と割り当て連番のペア）

## 手順

1. 渡された担当ファイルリストに基づき、各画像を Read で読み取り内容を確認する（**可能な限り並列で読み取る**）
2. 内容に基づいて、わかりやすいファイル名を決定する
   - 命名規則: `NNN_連番_説明.jpeg`（例: `004_01_paras_caught.jpeg`, `004_02_rival_battle.jpeg`）
   - 連番は呼び出し元から指定されたものを使う（ゼロ埋め2桁）
   - 説明は英語スネークケースで簡潔に
3. `images/NNN_エリア名/` ディレクトリを作成し、担当分のファイルをまとめて移動・リネームする（**1回の Bash で全ての mv を `&&` で連結して実行。`cd` は使わず、絶対パスで `mv` する**）
4. 処理結果を**連番順（＝時系列順）**で出力する

## 出力フォーマット

```
## 画像処理結果

| # | 元ファイル名 | 新ファイルパス | 説明 |
|---|---|---|---|
| 1 | IMG_001.jpeg | images/NNN_xxx/NNN_01_yyy.jpeg | 画像の内容説明 |
| 2 | IMG_002.jpeg | images/NNN_xxx/NNN_02_zzz.jpeg | 画像の内容説明 |

画像総数: N枚
```

## 注意事項

- 渡された担当ファイルのみ処理する。他のファイルは触らない
- サブディレクトリ内の既存画像は触らない
- ファイル名の説明部分は、レポートで `![説明]()` に使えるよう簡潔で内容がわかるものにする
- 担当ファイルが0件の場合は「処理対象の画像なし」と返す
