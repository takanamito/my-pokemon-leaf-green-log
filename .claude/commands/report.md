## レポート作成スキル

ポケモンリーフグリーン攻略のレポートを作成する。

## 手順

### 1. レポートファイルを作成

- `reports/` ディレクトリに連番のマークダウンファイルを作成
- 命名規則: `NNN_エリア名.md`（例: `004_mt_moon.md`）
- 既存レポートを確認して連番を決定: !`ls reports/`

### 2. レポートの構成

以下のセクションで構成する：

```
# NNN - タイトル
## 日時
## 現在地
## 攻略ログ（メインコンテンツ）
## パーティ編成（テーブル形式）
## 入手アイテム（テーブル形式）
## 次の目標
```

### 3. 攻略ログの書き方（最重要）

攻略ログがレポートの核。以下のルールを厳守する：

- **「戦略メモ」「会話サマリー」のような独立した見出しセクションは絶対に作らない**
- 戦略判断・Claude のミス・takanamito との会話は、**該当する出来事の直後に地の文として書く**
- 会話は口語調でそのまま記録し、実際の会話の温度感が伝わるように書く
- takanamito の発言は**太字**で強調する（例: takanamito が「**そこはたいあたりだろ**」と突っ込んだ）
- Claude のミスや判断ミスも隠さず記録する（動画の面白ポイントになる）
- 以下の要素を積極的にログ内に盛り込む：
  - 重要な意思決定の経緯（何を相談し、どういうやりとりを経て決まったか）
  - takanamito のツッコミや印象的なリアクション
  - Claude が間違えて takanamito に正された場面
  - 意見が分かれた場面や議論のポイント
  - 確率計算や豆知識など、会話中に出た雑学
- **参考スタイル**: `reports/004_mt_moon.md` を読んでスタイルを踏襲すること

### 4. 画像の整理

スクリーンショットが `images/` 直下にある場合：

1. `images/NNN_エリア名/` サブディレクトリを作成
2. Agent ツールで画像の内容を確認し、内容がわかるファイル名にリネームして移動
3. 命名規則: `NNN_説明.jpeg`（例: `004_paras_caught.jpeg`）
4. レポート内から `![説明](../images/NNN_エリア名/NNN_説明.jpeg)` で参照

### 5. README.md の更新

- `README.md` のレポート一覧テーブルに新しいレポートを追加する

### 6. 直前のレポートとの一貫性チェック

- 直前のレポートを読んで、パーティ編成やレベルの整合性を確認する
