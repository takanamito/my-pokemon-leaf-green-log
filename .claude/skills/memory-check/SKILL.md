## 指示前の自己チェッカー

行動を指示する前に、過去の会話との整合性を確認する。
Claude は自分の指示を忘れて同じ指示を繰り返したり、矛盾する指示を出すことがある。

**指示を出す前に、以下の5項目を順番にチェックする。**

---

## チェック1: 同じ指示を既に出していないか

技の指示、アイテムの使用、TMの使用など、**指示を出す前に会話の直近の流れを確認する**。

危険パターン:
- 「○○を使おう」→（結果報告）→ また「○○を使おう」
- 「○○を教えよう」→（結果報告）→ また「○○を教えよう」

## チェック2: アイテムの効果を正しく理解しているか

| カテゴリ | アイテム例 | 効果 |
|---------|----------|------|
| HP回復 | キズぐすり、いいキズぐすり | HPを回復 |
| 状態異常回復 | どくけし、まひなおし | 状態異常を治す |
| PP回復 | PPエイド、PPリカバー | **技のPPを回復（HPではない）** |
| PP全回復 | ピーピーマックス | 全技のPP全回復 |

不確かな場合は正直に「このアイテムの効果を確認させて」と言う。

## チェック3: 状況を正しく把握しているか

- 状態異常が発生したとき → **どちらのポケモンが状態異常になったか**を確認する
- 「まひになった」→ こちらか？相手か？ を明確にしてから指示を出す
- takanamito の報告をよく読み、思い込みで判断しない

## チェック4: ゲーム内の名称は正確か

- アイテム名、技名、ポケモン名は**第3世代（ファイアレッド・リーフグリーン）での正式名称**を使う
- 他世代の名称と混同しないこと（例: らせんのかせき → かいのカセキ）
- 不確かな場合は takanamito に画面の選択肢を確認してもらう

## チェック5: 技スロットの空きを確認したか

技を忘れさせる指示を出す前に、**そのポケモンが現在いくつ技を覚えているか確認する**。

- 4つ未満 → 忘れる必要なし（空きスロットに入る）
- 4つ → どの技を忘れるか判断が必要
- **このミスは2回やった（ゼニガメのからにこもる、パラスのTM09）。3回目は許されない**

## 過去のミスから学ぶ

具体的なミスの記録は `references/past-mistakes.md` にまとめてある。特に多いパターン:

- **同じ指示の繰り返し**: どくけし使用直後に再度「どくけしを使おう」、TM使用直後に再度「教えよう」
- **アイテム効果の誤認**: PPエイドを「HP60回復」と説明した（実際はPP回復）
- **状況の誤読**: こちらが麻痺したのに「相手に状態異常で捕まりやすい」と判断
- **名称の混同**: 第3世代では「かいのカセキ」なのに「らせんのかせき」と指示
