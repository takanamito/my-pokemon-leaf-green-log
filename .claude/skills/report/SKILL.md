## レポート作成スキル

ポケモンリーフグリーン攻略のレポートを作成する。
**サブエージェントを活用して画像処理と情報収集を並列化し、高速にレポートを仕上げる。**

---

## 手順

### 1. 準備

- `reports/` ディレクトリの既存レポートを確認して連番を決定: !`ls reports/`
- レポート番号 `NNN` とエリア名を決定

### 2. フェーズ1 — 情報収集（並列）

以下の2つを **同時に** Agent ツールで起動する（依存関係がないため並列実行可能）：

#### `report-image` エージェント（画像の内容確認 + リネーム + 移動）
- `.claude/agents/report-image.md` で定義
- `images/` 直下の画像を読み取り、内容に基づいたファイル名でリネーム・移動する
- プロンプトにレポート番号とエリア名を渡す

#### `report-consistency` エージェント（前回レポートの整合性チェック）
- `.claude/agents/report-consistency.md` で定義
- 直前のレポートからパーティ編成・アイテム・次の目標を抽出して返す

### 3. フェーズ2 — レポート本文作成（順次）

両エージェントの結果を使って、`reports/NNN_エリア名.md` を作成する。

#### レポートの構成

```
# NNN - タイトル
## 日時
## 現在地
## 攻略ログ（メインコンテンツ）
## パーティ編成（テーブル形式）
## 入手アイテム（テーブル形式）
## 次の目標
```

#### 画像の参照

`report-image` エージェントの結果（新ファイルパスと説明テキスト）を使って参照する：
`![説明](../images/NNN_エリア名/NNN_連番_説明.jpeg)`

**画像の配置ルール:**
- 画像は連番順（＝時系列順）に沿って配置する。連番が小さいものが先に来る
- 各画像は**その内容に関連する文章の直後**に置く。無関係な文章の下に画像を置かない
- 連番順に配置していけば、文章の流れと画像の流れが自然に一致する

#### 整合性チェック

`report-consistency` エージェントの結果を使い、以下を確認する：
- パーティ編成のレベルが前回から矛盾なく上がっているか
- 前回の「次の目標」と今回の内容が整合しているか

### 4. 攻略ログの書き方（最重要）

攻略ログがレポートの核。以下のルールを厳守する：

- **「戦略メモ」「会話サマリー」のような独立した見出しセクションは絶対に作らない**
- 戦略判断・Claude のミス・takanamito との会話は、**該当する出来事の直後に地の文として書く**
- 会話は口語調でそのまま記録し、実際の会話の温度感が伝わるように書く
- takanamito の発言は**太字**で強調する（例: takanamito が「**そこはたいあたりだろ**」と突っ込んだ）
- Claude のミスや判断ミスも隠さず記録する（動画の面白ポイントになる）
- 以下の要素を積極的に盛り込む：
  - 重要な意思決定の経緯
  - takanamito のツッコミや印象的なリアクション
  - Claude が間違えて正された場面
  - 意見が分かれた場面
  - 確率計算や豆知識など、会話中に出た雑学
- **参考スタイル**: `reports/004_mt_moon.md` を読んでスタイルを踏襲すること

### 5. フェーズ3 — 付随ファイル更新（並列）

レポート本文の完成後、以下の2つを **同時に** Agent ツールで起動する：

#### `report-readme` エージェント（README.md 更新）
- `.claude/agents/report-readme.md` で定義
- レポート番号・エリア名・ファイルパスを渡す

#### `report-pokedex` エージェント（ポケモン図鑑更新）
- `.claude/agents/report-pokedex.md` で定義
- 以下の情報を渡す：
  - レポート番号とファイルパス
  - パーティの最新情報（ポケモン名、レベル、技構成）
  - 新規捕獲・進化があれば、その情報とレポート内の見出しアンカー
  - 見かけたけど捕まえなかったポケモンの情報

#### ポケモン図鑑の更新ケース（エージェントに渡す情報の参考）

- **新規捕獲**: レポートに `### ○○捕獲` の見出しを書き、アンカーリンク付きで渡す
- **進化**: レポートに `### ○○進化` の見出しを書き、進化前後の情報を渡す
- **レベル・技変更**: パーティ編成テーブルの最新データを渡す
- **見かけたポケモン**: ポケモン名・場所・スルー理由を渡す
