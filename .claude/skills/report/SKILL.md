## レポート作成スキル

ポケモンリーフグリーン攻略のレポートを作成する。
**サブエージェントを活用して画像処理と情報収集を並列化し、高速にレポートを仕上げる。**

## 手順

### 1. 準備

- `reports/` ディレクトリの既存レポートを確認して連番を決定: !`ls reports/`
- レポート番号 `NNN` とエリア名を決定
- `images/NNN_エリア名/` サブディレクトリを作成

### 2. サブエージェントを並列起動

以下の3つのサブエージェントを **同時に** Agent ツールで起動する（依存関係がないため並列実行可能）：

#### サブエージェント A: 画像の内容確認

- `images/` 直下にあるスクリーンショットを **すべて読み取り**、各画像の内容を説明する
- 出力形式: 各画像について `{元ファイル名, 内容の説明}` のリストを返す

#### サブエージェント B: 画像ファイルのリネームと移動

- `images/` 直下にあるスクリーンショットを **すべて読み取り**、内容に基づいたファイル名を決定する
- 命名規則: `NNN_説明.jpeg`（例: `004_paras_caught.jpeg`）
- ファイルを `images/NNN_エリア名/` に移動・リネームする
- 出力形式: 各画像について `{元ファイル名, 新ファイルパス, 説明テキスト}` のリストを返す

#### サブエージェント C: 直前レポートの読み込みと整合性情報の収集

- 直前のレポート（最新の `reports/*.md`）を読み込む
- 以下の情報を抽出して返す：
  - パーティ編成（ポケモン名、レベル、技構成）
  - 入手済みアイテム
  - 現在地
  - 次の目標
- この情報はレポート作成時の整合性チェックに使う

### 3. レポートファイルを作成

サブエージェント B, C の結果を使って、`reports/NNN_エリア名.md` を作成する。

#### レポートの構成

```
# NNN - タイトル
## 日時
## 現在地
## 攻略ログ（メインコンテンツ）
## パーティ編成（テーブル形式）
## 入手アイテム（テーブル形式）
## 次の目標
```

#### 画像の参照

サブエージェント B の結果（新ファイルパスと説明テキスト）を使って、レポート内から画像を参照する：
`![説明](../images/NNN_エリア名/NNN_説明.jpeg)`

#### 整合性チェック

サブエージェント C の結果を使い、以下を確認する：
- パーティ編成のレベルが前回から矛盾なく上がっているか
- 前回の「次の目標」と今回の内容が整合しているか

### 4. 攻略ログの書き方（最重要）

攻略ログがレポートの核。以下のルールを厳守する：

- **「戦略メモ」「会話サマリー」のような独立した見出しセクションは絶対に作らない**
- 戦略判断・Claude のミス・takanamito との会話は、**該当する出来事の直後に地の文として書く**
- 会話は口語調でそのまま記録し、実際の会話の温度感が伝わるように書く
- takanamito の発言は**太字**で強調する（例: takanamito が「**そこはたいあたりだろ**」と突っ込んだ）
- Claude のミスや判断ミスも隠さず記録する（動画の面白ポイントになる）
- 以下の要素を積極的にログ内に盛り込む：
  - 重要な意思決定の経緯（何を相談し、どういうやりとりを経て決まったか）
  - takanamito のツッコミや印象的なリアクション
  - Claude が間違えて takanamito に正された場面
  - 意見が分かれた場面や議論のポイント
  - 確率計算や豆知識など、会話中に出た雑学
- **参考スタイル**: `reports/004_mt_moon.md` を読んでスタイルを踏襲すること

### 5. README.md の更新

- `README.md` のレポート一覧テーブルに新しいレポートを追加する
