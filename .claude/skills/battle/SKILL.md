## バトル技選択・先頭ポケモン決定スキル

バトルの意思決定を3つのフェーズで行う: **バトル開始** → **バトル中（技選択）** → **バトル後（次の準備）**

---

## フェーズ1: バトル開始 — 誰を出すか

### トレーナー戦・野生戦の先頭選択

1. 相手のポケモンのタイプを確認する
2. 育成対象を先頭に出せるか判断する（詳細は「育成対象の決定」を参照）
3. ライバル戦・ボス戦 → **エースを先頭にする**（育成より安全優先）

### 経験値分配の原則

**育成したいポケモンを先頭 → すぐエースに交代**が最も効率的。

- 育成ポケモン: ダメージを受けずに経験値がもらえる
- エースポケモン: 交代で出て敵を倒し、経験値がもらえる
- 逆パターン（エースで削って育成ポケモンに交代）は育成ポケモンがダメージを受けるリスクがあるので避ける

## フェーズ2: バトル中 — 技選択

### damage-calc エージェントへの委譲

技を推奨するときは、**`damage-calc` エージェント（`.claude/agents/damage-calc.md`）に計算が自動的に委譲される**。
自分で相性を推測して技を選ばないこと。過去のタイプ相性ミスはすべて自力計算が原因だった。

#### 流れ

1. バトルで技選択が必要になると、`damage-calc` エージェントが自動起動する
2. エージェントの結果（推奨技）をそのまま takanamito に伝える
3. 状態異常技を使うかどうかの判断だけこのスキルで行う（下記参照）

### 技選択の判断基準

1. `damage-calc` が返した**最も実効威力の高い技**を選ぶ
2. 実効威力が同じ場合は、命中率が高い方を選ぶ
3. PPが残り少ない場合は、2番目の候補を検討する
4. 同タイプの技が複数ある場合（例: からてチョップとけたぐり）→ **威力だけで比較する**。タイプ相性は同じなので「相性が違う」と説明しないこと

### 状態異常技の活用判断

状態異常技を使うかどうかは damage-calc ではなくこのスキルで判断する。

- 相手のレベルが高い / HP管理が厳しい → しびれごな等でまひにして安全に戦う
- 相手のレベルが低い / 余裕がある → 攻撃で倒す（状態異常技は不要）
- 捕獲時 → しびれごな・ねむりごなが最優先

### トレーナー戦の途中（相手が次のポケモンを出したとき）

- 新しく出てきたポケモンのタイプを確認してから技を指示する
- 有利なポケモンが控えにいるなら交代を検討する
- HPが少ないポケモンは無理させず交代する

## フェーズ3: バトル後 — 次の準備

**毎回のバトル後に**、以下を判断して宣言する。takanamito に聞かれる前に自分から宣言すること。

### 育成対象の決定

以下の優先順位で判断する。**単なるレベル差ではなく、攻略への貢献度で決める。**

1. **次のジムリーダーや直近のボス戦で活躍できるポケモンを最優先**
   - タイプ相性で有利なポケモン、または弱点を突ける技を持つポケモン
2. **今後のストーリーで長期的に戦力になるポケモン**
   - パーティの中核として使い続ける予定のポケモンを優先
3. **進化が近いポケモン**
   - あと数レベルで進化するなら優先度を上げる
4. **上記を満たす中で、レベルが低いポケモンを優先**
   - レベル差はあくまで同条件のポケモン間の判断材料

### 先頭ポケモンの決定

1. 次の相手（トレーナーの手持ち or エリアの野生ポケモン）のタイプを把握する
2. 育成対象を先頭に出せるか判断する:
   - 弱点を突かれない → **育成対象を先頭にして、すぐ交代 or そのまま戦わせる**
   - 弱点を突かれるが耐えられる → **育成対象を先頭に出してすぐ交代**（経験値は入る）
   - 一撃で倒される危険がある（レベル差+弱点） → **別のポケモンを先頭にする**
3. 育成対象が先頭に出せない場合 → 次に育成が必要なポケモンや、相手に有利なポケモンを先頭にする
4. **ライバル戦・ボス戦の直前 → エースを先頭にする**

### 回復判断の基準

- 先頭ポケモンのHPが**半分以下** → キズぐすり推奨
- 状態異常あり → 対応する回復アイテムを使う
- エースのHPが**3分の1以下** → 必ず回復してから進む
- 次がライバル戦・ボス戦 → 全員全快にする
- 回復アイテムの在庫が少ない → ポケモンセンターや船内の回復NPCを利用

## 回答フォーマット

### バトル後の先頭宣言

バトルが終わるたびに、この形式で宣言する。**省略せず毎回宣言すること。**

```
【次のバトル準備】
回復: ○○にキズぐすり / 回復不要 / ポケモンセンターに戻る
次の先頭: ○○（Lv○○）
理由: ○○（次のジムに有利 / 進化まであとLv○○ 等）
相手の想定: ○○タイプが多い / 次のトレーナーは○○を使う
作戦: ○○先頭→○○に交代 / そのまま○○で攻撃
```

### エリア移動時の先頭宣言

```
先頭: ○○
育成対象: ○○（理由: 次のジムに有利 / 進化が近い 等）
作戦: ○○先頭→○○に交代 / そのまま戦わせる
控え: ○○に交代する想定
注意: ○○（ライバル戦の可能性がある場合等）
```

## 技スロット管理

ポケモンは**最大4つ**の技を覚えられる。新しい技を覚えるときは:

1. **まず現在いくつ技を覚えているか確認する**
2. 4つ未満なら忘れさせる必要はない（空きスロットに入る）
3. 4つ覚えている場合のみ、どの技を忘れるか判断する

## 過去のミスから学ぶ

Claude は過去に多くのタイプ相性ミスや技選択ミスを犯している。具体的な記録は `references/past-mistakes.md` にまとめてある。

特に危険なパターン:
- **くさ技を虫・飛行タイプに撃つ** → 半減〜二重半減で大事故になった
- **かくとう技を飛行・虫タイプに撃つ** → 半減なのにばつぐんと判断した
- **タイプ一致ボーナスを忘れる** → 一致技の実効威力を過小評価して弱い技を推奨した
- **ポケモンのタイプを誤認する** → サンドをじめん/いわと思い込んだ（実際は純じめん）

これらのミスは `damage-calc` エージェントへの委譲で防げるはず。自力計算をしないこと。
