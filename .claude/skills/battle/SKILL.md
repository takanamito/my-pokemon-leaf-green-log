## バトル技選択・先頭ポケモン決定スキル

バトルの意思決定を3つのフェーズで行う: **バトル開始** → **バトル中（技選択）** → **バトル後（次の準備）**

---

## フェーズ1: バトル開始 — 誰を出すか

### 大原則: 安全第一、育成は余裕があるときだけ

**育成のために無駄なダメージを受けるのは本末転倒。** 回復アイテムを消費したりポケモンセンターに戻る羽目になれば、育成どころか攻略が遅れる。

### 先頭ポケモンの決定フロー（この順序で判断する）

1. **ライバル戦・ボス戦か？** → YES → エースを先頭。育成は考えない
2. **相手のタイプを確認**（野生なら出現テーブル、トレーナーなら手持ち）
3. **タイプ相性で有利なポケモンがいるか？** → YES → そのポケモンを先頭にする
4. **有利なポケモンが育成対象と一致するか？** → YES → 最高の状況。そのまま戦わせる
5. **有利なポケモンが育成対象と違う場合** → 有利なポケモンを先頭にする（育成より被ダメ軽減を優先）
6. **等倍同士で明確な有利不利がない場合** → ここで初めて育成対象を先頭に出す余地がある

### 育成対象を先頭に出してよい条件（すべて満たすこと）

- 相手から弱点を突かれない
- 育成対象のレベルが相手より3以上低くない
- 育成対象のHPが7割以上ある
- 上記を満たさない場合は、素直に有利なポケモンかエースを出す

### 経験値分配の原則

育成対象が安全に先頭に出せる場合のみ、**育成対象を先頭 → すぐ交代**で経験値を分配する。

- 育成ポケモン: ダメージを受けずに経験値がもらえる
- エースポケモン: 交代で出て敵を倒し、経験値がもらえる
- **やってはいけないこと**: 弱点を突かれる育成ポケモンを「交代するから大丈夫」と先頭に出すこと。先制技や急所で落ちるリスクがある

## フェーズ2: バトル中 — 技選択

### バトルチートシートの活用

`prep` スキル実行時にバックグラウンドで `battle-prep` エージェントが生成したチートシートがコンテキストにある場合、**まずチートシートを参照して即答する**。チートシートに載っている敵ポケモンなら計算不要で推奨技を答えられる。

チートシートにない敵（想定外のポケモン）が出た場合のみ、以下の手順で計算する。

### 実効威力の計算（チートシートにない場合）

技を推奨するときは、以下の相性表と計算式を使って**自分で直接計算する**。
過去のミスは推測が原因だったので、**必ず下の相性表を見て**計算すること。

#### 計算式

```
実効威力 = 技の威力 × タイプ一致ボーナス × タイプ相性
- タイプ一致ボーナス: 技のタイプが味方ポケモンのタイプと一致 → ×1.5、不一致 → ×1.0
- タイプ相性: ばつぐん ×2.0、等倍 ×1.0、いまひとつ ×0.5、こうかなし ×0
- 複合タイプ: 各タイプの倍率を掛け合わせる（例: いわ/じめん に みず → 2.0 × 2.0 = 4.0）
- 連続技（タネマシンガン等）: 最低回数（2回）での合計威力で計算
```

### 技選択の判断基準

1. **必ず全技の実効威力を計算してから選ぶ**（感覚で選ばない）
2. **最も実効威力の高い技**を選ぶ
3. 実効威力が同じ場合は、命中率が高い方を選ぶ
4. PPが残り少ない場合は、2番目の候補を検討する
5. 同タイプの技が複数ある場合（例: からてチョップとけたぐり）→ **威力だけで比較する**。タイプ相性は同じなので「相性が違う」と説明しないこと

### よくある技選択ミスの防止

- **タイプ一致ボーナスの見落とし**: 等倍でもタイプ一致技（×1.5）は、不一致のばつぐん技と同等以上になることがある。必ず計算する
- **威力の低い技を安易に選ばない**: 「こうかばつぐんだから」という理由だけで低威力技を選ぶと、タイプ一致の等倍技より弱いことがある（例: 威力40のばつぐん技=実効80 < タイプ一致60の等倍技=実効90）

### 状態異常技の活用判断

- 相手のレベルが高い / HP管理が厳しい → しびれごな等でまひにして安全に戦う
- 相手のレベルが低い / 余裕がある → 攻撃で倒す（状態異常技は不要）
- 捕獲時 → しびれごな・ねむりごなが最優先

### トレーナー戦の途中（相手が次のポケモンを出したとき）

- 新しく出てきたポケモンのタイプを確認してから技を指示する
- **今のポケモンが不利（弱点を突かれる）→ 即交代**。「あと1発で倒せるかも」で居座らない
- 有利なポケモンが控えにいるなら交代する
- HPが半分以下のポケモンは無理させず交代する

### 相手ポケモンのタイプが不明な場合

`.claude/skills/type/pokemon_types.md` を読んでタイプを確認すること。推測でタイプを答えない。

## フェーズ3: バトル後 — 次の準備

**毎回のバトル後に**、以下を判断して宣言する。takanamito に聞かれる前に自分から宣言すること。

### 育成対象の決定

以下の優先順位で判断する。**単なるレベル差ではなく、攻略への貢献度で決める。**

1. **次のジムリーダーや直近のボス戦で活躍できるポケモンを最優先**
   - タイプ相性で有利なポケモン、または弱点を突ける技を持つポケモン
2. **今後のストーリーで長期的に戦力になるポケモン**
   - パーティの中核として使い続ける予定のポケモンを優先
3. **進化が近いポケモン**
   - あと数レベルで進化するなら優先度を上げる
4. **上記を満たす中で、レベルが低いポケモンを優先**
   - レベル差はあくまで同条件のポケモン間の判断材料

### 先頭ポケモンの決定

**フェーズ1と同じ判断フローに従う。** 育成のために被ダメを許容しない。

1. 次の相手のタイプを把握する
2. **タイプ相性で有利なポケモンを先頭にする**（これが基本）
3. 有利なポケモンが育成対象と一致する → そのまま戦わせる（ベストケース）
4. 相性が等倍同士で差がない場合のみ、育成対象を先頭にする余地がある
5. **ライバル戦・ボス戦の直前 → エースを先頭にする**

**以前のミス**: 「弱点を突かれるが耐えられる → 育成対象を先頭に出してすぐ交代」は禁止。先制技・急所・連続技で想定外のダメージを受ける

### 回復判断の基準

- 先頭ポケモンのHPが**半分以下** → キズぐすり推奨
- 状態異常あり → 対応する回復アイテムを使う
- エースのHPが**3分の1以下** → 必ず回復してから進む
- 次がライバル戦・ボス戦 → 全員全快にする
- 回復アイテムの在庫が少ない → ポケモンセンターや船内の回復NPCを利用

## 回答フォーマット

### バトル後の先頭宣言

バトルが終わるたびに、この形式で宣言する。**省略せず毎回宣言すること。**

```
【次のバトル準備】
回復: ○○にキズぐすり / 回復不要 / ポケモンセンターに戻る
次の先頭: ○○（Lv○○）
理由: ○○（次のジムに有利 / 進化まであとLv○○ 等）
相手の想定: ○○タイプが多い / 次のトレーナーは○○を使う
作戦: ○○先頭→○○に交代 / そのまま○○で攻撃
```

### エリア移動時の先頭宣言

```
先頭: ○○
育成対象: ○○（理由: 次のジムに有利 / 進化が近い 等）
作戦: ○○先頭→○○に交代 / そのまま戦わせる
控え: ○○に交代する想定
注意: ○○（ライバル戦の可能性がある場合等）
```
