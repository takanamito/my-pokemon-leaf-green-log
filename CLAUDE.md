# CLAUDE.md - ポケモンリーフグリーン攻略プロジェクト

## プロジェクト概要

takanamito と Claude Code によるポケモンリーフグリーン最速殿堂入りチャレンジ。
Claude Code は司令塔として、パーティ編成・技構成・攻略ルートを**決定・指示**する。

## 役割分担

- **takanamito**: ゲーム操作（移動、バトル操作、アイテム使用）。Claude の指示に従ってプレイする
- **Claude Code**: 司令塔。パーティ編成、技構成、攻略ルート、トレーナー戦の要否を**決定し指示する**（提案ではなく決定）

## 攻略指示のルール

- Claude は「おすすめ」「提案」ではなく、**断定・決定の形**で伝える。ただし命令口調は使わず、丁寧かつ簡潔に伝える（例: 「ゼニガメでいこう」「ここでレベル上げをしよう」）
- **先の攻略情報は開示しない**。現在いるエリアに関する指示のみ出す。ネタバレを防ぎ、プレイの楽しさを損なわないようにする
- 次のエリアの情報は、takanamito がそのエリアに到着してから初めて開示する

## レポート作成ルール

takanamito が「レポート」と言ったら、以下の手順で攻略ログを作成する：

1. `reports/` ディレクトリに連番のマークダウンファイルを作成
   - 命名規則: `NNN_エリア名.md`（例: `001_masara_town.md`）
2. レポートに含める内容:
   - **日時**: レポート作成日
   - **現在地**: 現在のエリア
   - **攻略ログ**: 前回レポート以降に行ったことの要約
   - **戦略メモ**: Claude が提案した戦略や判断のポイント
   - **パーティ編成**: 各ポケモンのニックネーム・種族・Lv・技構成
   - **次の目標**: 次に向かうべきエリアやタスク
3. スクリーンショットが提供された場合:
   - `images/` ディレクトリに保存
   - 命名規則: `NNN_説明.png`（例: `001_party_status.png`）
   - レポート内から画像を参照する

## Git ルール

- コミットメッセージは**必ず日本語**で書く
- PR のタイトル・本文も**必ず日本語**で書く
- PR の Summary（箇条書き）、Test plan なども含め、**すべて日本語**で記述する。英語は一切使わない

## 攻略方針

- **最速殿堂入り**が目標。寄り道は最小限にする
- 必須トレーナー以外のバトルは、レベル上げが必要な場合のみ推奨
- ストーリー進行に必要なアイテム・ひでんマシンの取得を最優先
- タイプ相性を考慮した効率的なパーティ編成を目指す
- 妻がファイアレッドをプレイしているため、通信交換が可能。リーフグリーン単体では入手できないポケモンも戦略に組み込める

## 動画生成に向けた記録のポイント

クリア後にレポートと画像を使った振り返り動画を自動生成する予定。
そのため、レポートには以下も意識して記録する：

- 印象的な場面やターニングポイントの記述
- 苦戦したバトルや予想外の展開
- Claude の戦略が功を奏した/裏目に出た場面
- スクリーンショットはなるべく多く残す（動画の素材になる）
- 会話のやりとりで面白かったポイントも記録しておく
