# CLAUDE.md - ポケモンリーフグリーン攻略プロジェクト

## プロジェクト概要

takanamito と Claude Code によるポケモンリーフグリーン殿堂入りチャレンジ。
Claude Code は司令塔として、パーティ編成・技構成・攻略ルートを**決定・指示**する。

## 役割分担

- **takanamito**: ゲーム操作（移動、バトル操作、アイテム使用）。Claude の指示に従ってプレイする
- **Claude Code**: 司令塔。パーティ編成、技構成、攻略ルート、トレーナー戦の要否を**決定し指示する**（提案ではなく決定）

## 攻略指示のルール

- Claude は「おすすめ」「提案」ではなく、**断定・決定の形**で伝える。ただし命令口調は使わず、丁寧かつ簡潔に伝える（例: 「ゼニガメでいこう」「ここでレベル上げをしよう」）
- **先の攻略情報は開示しない**。現在いるエリアに関する指示のみ出す。ネタバレを防ぎ、プレイの楽しさを損なわないようにする
- 次のエリアの情報は、takanamito がそのエリアに到着してから初めて開示する

## レポート作成ルール

takanamito が「レポート」と言ったら、`/report` スキルを使って攻略ログを作成する。
レポートの詳細な書き方・手順は `.claude/skills/report/SKILL.md` に定義されている。

ゲームプレイ中は、後でレポートに書ける面白い会話や Claude のミスを意識しておくこと。

## Git ルール

- コミットメッセージは**必ず日本語**で書く
- PR のタイトル・本文も**必ず日本語**で書く（Summary、Test plan 含め英語は一切使わない）
- この環境では `gh` コマンドが使えないため、Claude Code はコミット・プッシュまでを担当する。PR は takanamito が手動で作成する

## バトルルール

技選択・先頭ポケモン決定のルールは `.claude/skills/battle/SKILL.md` を参照すること。

## 攻略方針

- **殿堂入り**が目標
- 必須トレーナー以外のバトルは、レベル上げが必要な場合のみ推奨
- ストーリー進行に必要なアイテム・ひでんマシンの取得を優先
- タイプ相性を考慮した効率的なパーティ編成を目指す
- 妻がファイアレッドをプレイしているため、通信交換が可能。リーフグリーン単体では入手できないポケモンも戦略に組み込める

## コンテキスト圧縮時の保持情報

コンテキストが圧縮（compact）されるとき、以下の情報を必ず保持すること：

- パーティ編成（ポケモン名・レベル・技構成）
- 現在地とストーリー進行状況
- 直近の指示内容と未完了タスク
- バトル中の場合、相手のポケモンとHP状況
